# workflows

Reusable workflow patterns and process automation skills.

## Skills

| Skill | Description |
|-------|-------------|
| **changelog-generator** | Automatically creates user-facing changelogs from git commits by analyzing history and categorizing changes |
| **code-review** | Mandatory code reviews via /code-review before commits and deploys |
| **commit-hygiene** | Atomic commits, PR size limits, commit thresholds, stacked PRs |
| **dispatching-parallel-agents** | Use when facing 2+ independent tasks that can be worked on without shared state or sequential dependencies |
| **finishing-a-development-branch** | Guides completion of development work by presenting structured options for merge, PR, or cleanup |
| **iterative-development** | Ralph Wiggum loops - self-referential TDD iteration until tests pass |
| **meeting-insights** | Analyzes meeting transcripts and recordings to uncover behavioral patterns and communication insights |
| **preserving-productive-tensions** | Recognize when disagreements reveal valuable context, preserve multiple valid approaches |
| **product-manager** | PM skill - diagnoses SaaS metrics, critiques PRDs, plans roadmaps, runs discovery, coaches PM career transitions |
| **session-management** | Context preservation, tiered summarization, resumability |
| **team-coordination** | Multi-person projects - shared state, todo claiming, handoffs |
